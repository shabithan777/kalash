## Описание проделанной работы
Реализована оптимизированная проверка: число проверяется на делимость только на уже найденные простые числа.
Проверка останавливается, когда делитель превышает квадратный корень из проверяемого числа.
Создана внешняя функция prime_generator(), внутри которой определены:

Список primes для хранения найденных простых чисел.

Переменная num = 2 — текущее проверяемое число.

Внутренняя функция is_prime(n) для проверки простоты.

Внутренняя функция get_next(), которая возвращает следующее простое число и обновляет состояние.
Проведена проверка работы замыкания: последовательные вызовы возвращают корректные простые числа.

## Задание 2. (Декоратор, ограничивающий время выполнения функции)
Постановка задачи
Реализовать декоратор timeout(seconds), который ограничивает время выполнения функции. Если функция выполняется дольше указанного количества секунд, выполнение должно прерываться с генерацией исключения TimeoutError.

Выполненная работа
Изучение теоретического материала

Рассмотрена концепция декораторов в Python: функции, принимающие другую функцию и возвращающие её обёртку.

Изучена работа с потоками (threading) для асинхронного контроля времени выполнения.

Выбор подхода

Выбран метод с использованием потока-демона (daemon thread), который запускает целевую функцию и ожидает её завершения не дольше заданного времени.

Реализация декоратора

Создана функция timeout(seconds), возвращающая декоратор.

Во внутренней функции-обёртке:

Создаются контейнеры result и error для передачи данных из потока.

Запускается поток с целевой функцией.

Поток ожидает завершения не дольше seconds секунд (thread.join(seconds)).

Если поток всё ещё активен, генерируется TimeoutError.

Если в потоке возникло исключение, оно пробрасывается в вызывающий код.

Устранение проблем совместимости

В среде Skulpt (онлайн-редактор) модуль functools не поддерживается, поэтому из кода убрано использование @functools.wraps.

Для максимальной совместимости предложена альтернативная реализация без потоков (с передачей лимита времени в аргументы функции).

Тестирование

Созданы тестовые функции с разным временем выполнения (1 секунда и 5 секунд).

Проверено, что быстрая функция успешно завершается, а медленная — генерирует исключение TimeoutError.
